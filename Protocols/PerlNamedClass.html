  • init(method:args:perl:) Extension method

    Creates a new object by calling its Perl constructor.

    The recomended way is to wrap this constructor invocations by implementing more concrete initializers which hide Perl method calling magic behind.

    Let’s imagine a class:

    final class URI : PerlObject, PerlNamedClass {
    static let perlClassName = "URI"

    convenience init(_ str: String) throws {
        try self.init(method: "new", args: [str])
    }
}

    Then Swift expression:

    let uri = URI("https://my.mail.ru/music")

    will be equal to Perl:

    my $uri = URI->new("https://my.mail.ru/music")

    Parameter

    Parameter method: A name of the constuctor. Usually it is new.

    Parameter

    Parameter args: Arguments to pass to constructor.

    Declaration

    Swift

    public init(method: String, args: [PerlSvConvertible?], perl: UnsafeInterpreterPointer = UnsafeInterpreter.current) throws

  • call(method:args:perl:) Extension method

    Calls Perl method on the class specified by perlClassName attribute.

    Arguments of the call will be automagically converted to mortalized Perl scalar values with the lifetime of the scope of this call. The similar thing will happen to Perl return values: they will be destroyed before the call returns (but after conversion to Swift values was done).

    Parameter

    Parameter name: A name of the method to call.

    Parameter

    Parameter args: A collection of arguments.

    Returns

    Return value of the call.

    Declaration

    Swift

    public static func call<C : Collection>(method: String, args: C, perl: UnsafeInterpreterPointer = UnsafeInterpreter.current) throws -> Void
		where C.Iterator.Element == PerlSvConvertible?
